# PDF双面打印延迟控制脚本

> **版本**: v0.3  
> 专为 Brother MFC-7480D 等老旧打印机设计的智能双面打印解决方案  
> **彻底解决连续双面打印卡纸问题**  
> 支持 CLI 命令行 + GUI 图形界面双模式  
> 支持文件/文件夹批量打印、份数设置、页面范围选择

---

## 📋 解决的问题

### 核心问题
**Brother MFC-7480D 打印机存在一个严重的硬件 Bug：连续双面打印时必然在第二页卡纸！**

具体表现为：
- ✗ 打印机的自动双面功能只能一页一页地打印双面
- ✗ 只要连续发送两个或更多双面打印任务，第二页必定卡纸
- ✗ 无法通过驱动设置或固件更新解决
- ✗ 打印多页文档时需要人工干预，每打印一页就要等待再继续

### 其他类似问题
许多老旧型号的打印机也存在类似问题：
- 打印机处理速度慢，连续发送任务时页面读取失败
- 机械进纸机构老化，需要间隔时间避免卡纸
- 需要手动分批打印，操作繁琐且低效

## 💡 设计原理

### Brother MFC-7480D 卡纸问题分析

**问题根源**：
- 打印机的双面打印机械结构存在设计缺陷
- 当第一页双面打印完成后，如果立即接收第二个双面任务，进纸机构还没有完全复位
- 导致第二页进纸时卡在半路，触发卡纸错误
- **这是硬件层面的时序问题，无法通过驱动或设置修复**

**解决思路**：
既然硬件无法修复，就从软件层面控制打印时序：
1. **强制延迟**：在两个双面打印任务之间插入足够的等待时间
2. **分批发送**：将大文档拆分成多个小任务，避免打印机自动连续处理
3. **状态同步**：确保打印机完全完成当前任务后再发送下一个

### 工作流程

```
双面模式 (long/short):
  原始PDF → 按双面分割 → 逐批打印 → 延迟等待 → 下一批 → 清理
  (N页)    (N/2个文件)   (2页/次)   (20秒)     (循环)   (自动)
                                      ↑
                              关键：给打印机足够的
                              机械复位时间

单面模式 (none):
  原始PDF → 直接打印整个文件 → 完成
  (N页)     (一次发送)        (无延迟, 无拆分)
```

### 技术要点

1. **智能分割**: 双面模式下使用 `pypdf` 库按页拆分 PDF，每 2 页一组（对应一张纸的正反面）；单面模式不拆分
2. **多引擎打印**: 支持 SumatraPDF（推荐）、Acrobat Reader、系统默认三种打印引擎，自动选择最优方案
3. **延迟控制**: 仅双面模式需要延迟，每批打印后强制等待 20 秒（针对 MFC-7480D 优化），确保打印机机械结构完全复位；单面模式自动跳过延迟
4. **双面配置**: 自动配置打印机为双面打印模式（长边/短边可选），SumatraPDF 引擎通过打印参数控制
5. **自动清理**: 临时文件存放在系统临时目录，打印完成后自动删除
6. **份数控制**: 支持 1-99 份打印，默认逐份打印（先打完完整一份再打下一份）
7. **页面范围**: 支持全部/奇数页/偶数页/自定义范围（如 `1-5,8,10-12`）

### 打印引擎系统

脚本支持三种打印引擎，按优先级自动选择（可通过 `-e`/`--engine` 参数手动指定）：

| 引擎 | 参数值 | 缩放控制 | 双面控制 | 说明 |
|------|--------|----------|----------|------|
| **SumatraPDF** | `sumatra` | `-print-settings fit` | `-print-settings duplexlong/short/simplex` | 推荐，同步打印，精确控制，不需要 DEVMODE 配置 |
| **Acrobat Reader** | `acrobat` | 隐式（Acrobat 默认行为） | DEVMODE 配置 | 传统引擎，异步打印 |
| **系统默认** | `shell` | 取决于系统程序 | DEVMODE 配置 | 降级方案，使用系统关联程序 |

**自动选择逻辑** (`--engine auto`，默认):
1. 首先检查 `vendor/SumatraPDF.exe`（打包内置的便携版）
2. 然后检查系统 PATH 和常见安装路径中的 SumatraPDF
3. 如果未找到 SumatraPDF，查找 Acrobat Reader
4. 如果都未找到，使用 ShellExecute

**SumatraPDF 便携版**:
- 已内置在打包后的 `vendor/` 目录中，无需额外安装
- 从 [SumatraPDF 官网](https://www.sumatrapdfreader.org/download-free-pdf-viewer) 下载
- 便携版为单个 exe，无需安装，不写注册表

### 为什么 20 秒？

经过实际测试：
- **15 秒**：已测试连续打印 81 张双面（161 页 PDF）无卡纸
- **20 秒**：更保险，适合打印机状态不佳时使用（★ 默认推荐）
- **10 秒**：仍有小概率卡纸

## 🚀 使用方法

### CLI 命令行模式

#### 基本用法
```bash
# 打印单个 PDF（默认打印机，20秒延迟，长边翻转双面）
python cli_app.py document.pdf

# 指定延迟时间（仅双面模式生效）
python cli_app.py document.pdf -d 25

# 指定打印机（支持模糊匹配）
python cli_app.py document.pdf -p Brother

# 按编号选择打印机
python cli_app.py document.pdf -p 1

# 短边翻转双面打印
python cli_app.py document.pdf --duplex short

# 单面打印（无需延迟，直接打印整个文件）
python cli_app.py document.pdf --duplex none

# 打印文件夹内所有 PDF
python cli_app.py "D:\Documents\PDFs"

# 打印 3 份（逐份打印）
python cli_app.py document.pdf -n 3

# 打印奇数页
python cli_app.py document.pdf -r odd

# 打印指定页面范围
python cli_app.py document.pdf -r "1-5,8,10-12"

# 指定打印引擎
python cli_app.py document.pdf -e sumatra
```

#### 辅助命令
```bash
# 列出所有可用打印机（带编号）
python cli_app.py -l

# 查看指定打印机当前配置
python cli_app.py -i -p Brother

# 查看默认打印机配置
python cli_app.py -i

# 查看完整帮助
python cli_app.py --help
```

#### 完整参数说明
```
位置参数:
  pdf_file              要打印的 PDF 文件路径，或包含 PDF 的文件夹路径

可选参数:
  -h, --help            显示帮助信息并退出
  -d, --delay N         每批打印后的延迟秒数（默认: 20），仅双面模式生效，单面模式自动跳过
  -n, --copies N        打印份数（默认: 1，范围: 1-99）
  -r, --pages RANGE     页面范围（默认: all）
                         all=全部, odd=奇数页, even=偶数页, 或自定义如 "1-5,8,10-12"
  -p, --printer NAME    打印机名称（支持模糊匹配如 'Brother'；或编号如 '1'）
  -e, --engine {auto,sumatra,acrobat,shell}
                        打印引擎（默认: auto）
                         auto: 自动选择 (SumatraPDF > Acrobat > Shell)
                         sumatra: 强制使用 SumatraPDF
                         acrobat: 强制使用 Acrobat Reader
                         shell: 使用系统默认程序
  -l, --list            列出所有可用打印机
  -i, --info            显示打印机当前配置信息
  --duplex {long,short,none}
                        双面打印模式: long=长边翻转(默认), short=短边翻转, none=单面
  --keep-temp           保留临时文件（用于调试）
  --no-config           跳过自动双面打印配置
```

#### 打印机选择方式
脚本支持三种打印机指定方式，解决不同系统下打印机名称差异问题：

1. **模糊匹配**：输入打印机名称的一部分，自动匹配
   ```bash
   python cli_app.py doc.pdf -p Brother     # 匹配所有含"Brother"的打印机
   python cli_app.py doc.pdf -p MFC         # 匹配所有含"MFC"的打印机
   ```
   如果匹配到多个，会进入交互式选择。

2. **数字编号**：使用 `-l` 查看编号后直接指定
   ```bash
   python cli_app.py -l                     # 查看编号
   python cli_app.py doc.pdf -p 1           # 选择第1号打印机
   ```

3. **交互式选择**：不指定 `-p` 参数时，会提示选择
   ```
   当前默认打印机: Brother MFC-7480D Printer
   使用默认打印机? (回车=是, n=选择其他):
   ```

#### 界面操作步骤：
1. **选择 PDF 文件或文件夹**
   - 点击「浏览文件...」选择单个 PDF
   - 点击「浏览文件夹...」选择包含 PDF 的文件夹
   - 或直接拖拽文件/文件夹到窗口（需安装 `tkinterdnd2`）

2. **选择打印机**
   - 从下拉框选择打印机
   - 可点击「刷新」重新加载打印机列表
   - 点击「查看信息」查看打印机当前配置

3. **设置打印份数**
   - 使用 Spinbox 设置 1-99 份
   - 默认逐份打印（先打完完整一份再打下一份）

4. **选择双面打印模式**
   - 长边翻转：适合竖向文档，翻页像书本
   - 短边翻转：适合横向文档，翻页像记事本
   - 单面：仅单面打印（自动禁用延迟和页面范围自定义）

5. **设置页面范围**（仅单个 PDF 文件时可编辑；选择文件夹时自动禁用）
   - 全部：打印所有页面
   - 奇数页：打印 1,3,5,... 页
   - 偶数页：打印 2,4,6,... 页
   - 自定义：输入页面范围，如 `1-5,8,10-12`

6. **设置延迟间隔**（仅双面模式可编辑；单面模式自动禁用）
   - 双面模式：建议 15-20 秒（Brother MFC-7480D）
   - 单面模式：无延迟

7. **选择打印引擎**
   - auto：自动选择（默认）
   - sumatra：强制使用 SumatraPDF
   - acrobat：强制使用 Acrobat Reader
   - shell：使用系统默认程序

8. **点击「开始打印」**
   - 进度条和日志区实时显示打印进度
   - 可随时点击「取消打印」中断任务

> **提示**:
> - 将鼠标悬停在任何参数控件上，会显示该参数的用途说明（Tooltip）。
> - 选择「单面」时，延迟间隔和页面范围（自定义）会自动禁用。
> - 选择文件夹时，页面范围强制为「全部」并禁用。

## ⚙️ 配置建议

### 延迟时间设置
> **注意**: 延迟仅在双面打印模式（长边/短边翻转）下生效。单面打印模式自动跳过延迟，直接打印整个文件。

不同打印机建议的延迟时间：

| 打印机型号 | 建议延迟 | 说明 |
|-----------|---------|------|
| **Brother MFC-7480D** | **20 秒** | **⚠️ 专门针对连续双面卡纸 Bug 优化<br>15 秒已测试 81 张双面无卡纸<br>20 秒更保险（★ 默认推荐）** |
| 激光打印机（高速） | 10-15 秒 | 处理速度较快 |
| 喷墨打印机（低速） | 20-30 秒 | 处理速度较慢 |
| 网络打印机 | 20-25 秒 | 考虑网络传输延迟 |
| 其他老旧打印机 | 15-25 秒 | 根据实际情况调整 |

### Brother MFC-7480D 特别说明
如果你的打印机有连续双面卡纸问题：
1. **首次使用**：建议从 20 秒开始测试
2. **稳定后**：可以逐步降低到 15 秒
3. **打印机老化**：如 15 秒仍偶尔卡纸，增加到 20-25 秒
4. **重要文档**：宁可慢一点，使用 20 秒确保万无一失

**实测数据**：
- ✅ 20 秒间隔：成功打印 161 页 PDF（81 张双面），0 次卡纸
- ✅ 15 秒间隔：成功打印 161 页 PDF（81 张双面），0 次卡纸
- ✗ 10 秒间隔：偶尔会在第 10-20 张时卡纸

### 双面打印模式
| 模式 | 参数 | 说明 |
|------|------|------|
| **长边翻转** | `--duplex long`（默认） | 适合竖向文档，翻页像书本，需要延迟控制 |
| **短边翻转** | `--duplex short` | 适合横向文档，翻页像记事本，需要延迟控制 |
| **单面** | `--duplex none` | 仅单面打印，不拆分 PDF，无延迟 |

### 页面范围说明
| 选项 | 参数值 | 说明 |
|------|--------|------|
| **全部** | `all`（默认） | 打印所有页面 |
| **奇数页** | `odd` | 打印 1,3,5,... 页 |
| **偶数页** | `even` | 打印 2,4,6,... 页 |
| **自定义** | `1-5,8,10-12` | 打印指定页面范围（逗号分隔，支持连字符表示范围） |

> **注意**:
> - 页面范围在 PDF 拆分**之前**过滤，不影响拆分逻辑
> - 选择文件夹时，页面范围强制为「全部」并禁用（文件夹内可能包含多个 PDF，每个 PDF 的页面范围不同）
> - 自定义范围示例：`1-5,8,10-12` 表示打印第 1,2,3,4,5,8,10,11,12 页

## 🔧 故障排除

### 问题 1: Brother MFC-7480D 仍然卡纸
**可能原因**：延迟时间不够、打印机机械部件老化、纸张质量问题

**解决方案**：
1. **增加延迟时间**：将延迟从 20 秒增加到 25 秒
2. **检查纸张**：使用优质打印纸，避免受潮或卷曲的纸张
3. **清洁打印机**：清理进纸辊和双面单元
4. **小批量测试**：先打印 10 页测试，确认稳定后再打印大文档

### 问题 2: 找不到打印机
**解决方案**：
```bash
# 列出所有可用打印机及编号
python cli_app.py -l
```
使用编号或更精确的名称片段指定打印机。注意不同系统下打印机名称可能不同。

### 问题 3: 打印失败或跳页
**解决方案**：增加延迟时间（`-d` 参数），给打印机更多处理时间。

### 问题 4: 打印引擎相关
**说明**: 脚本默认使用 `auto` 模式自动选择打印引擎（优先级: SumatraPDF > Acrobat > Shell）。

**解决方案**:
- 打包后的 exe 已内置 SumatraPDF 便携版，通常无需额外配置
- 如果使用 Python 源码运行，请下载 [SumatraPDF 便携版](https://www.sumatrapdfreader.org/download-free-pdf-viewer) 放入 `vendor/` 目录
- 也可安装 Adobe Acrobat Reader 作为备用引擎
- 使用 `--engine` 参数（CLI）或下拉框（GUI）手动指定引擎
- 如果所有引擎都不可用，会降级使用系统默认打印程序

### 问题 5: 临时文件
**说明**：临时文件存放在系统临时目录（`%TEMP%\pdf_duplex_*`），打印完成后自动删除。如需调试，使用 `--keep-temp` 参数（CLI）或勾选「保留临时文件」（GUI）保留临时文件。

### 问题 6: GUI 无法启动
**解决方案**：
- 确保系统安装了 tkinter（大多数 Python 安装已内置）
- Windows 系统通常无需额外安装
- 如果使用了拖拽功能，请确保已安装 `tkinterdnd2`：
  ```bash
  pip install tkinterdnd2
  ```

### 问题 7: 页面范围不生效
**可能原因**：页面范围过滤在 PDF 拆分之前执行，如果页面范围格式不正确，会回退到「全部」模式。

**解决方案**：
- 检查页面范围格式是否正确（如 `1-5,8,10-12`）
- 页面编号从 **1** 开始（不是从 0 开始）
- 如果页面范围解析失败，会打印所有页面并输出警告信息

## 📌 注意事项

1. **Brother MFC-7480D 用户必读**：
   - ⚠️ 这个脚本是专门为解决连续双面卡纸问题设计的
   - 20 秒延迟是经过实际测试的最佳值（已验证 161 页无卡纸）
   - 如果你的打印机老化严重，建议使用 25 秒
   - 不要试图缩短延迟时间，否则会重新出现卡纸问题

2. **打印引擎**: 推荐使用 SumatraPDF（默认自动选择），它支持显式控制缩放和双面模式，且同步打印更可靠。打包版已内置 SumatraPDF 便携版。

3. **延迟时间**: 首次使用建议从较大的延迟开始（如 20 秒），观察效果后再调整

4. **打印队列**: 确保打印机队列中没有积压的任务

5. **纸张充足**: 确保纸盒中有足够的纸张（双面打印至少计算页数÷2 的张数）

6. **中断恢复**: 如果中途中断（Ctrl+C 或 GUI 取消），已发送的打印任务仍会继续执行

7. **时间成本**: 大文档打印需要较长时间，161 页的文档约需 20-25 分钟（20 秒延迟 × 81 批），请做好心理准备

8. **成功率**: 严格按照推荐配置使用，成功率可达 100%

9. **份数打印**: 默认逐份打印（先打完完整一份再打下一份），确保文档顺序正确

10. **页面范围**: 选择文件夹时，页面范围强制为「全部」（文件夹内可能包含多个 PDF）

# PDF双面打印延迟控制脚本

> 专为Brother MFC-7480D等老旧打印机设计的智能双面打印解决方案  
> **彻底解决连续双面打印卡纸问题**  
> 支持 CLI 命令行 + GUI 图形界面双模式

## 📋 解决的问题

### 核心问题
**Brother MFC-7480D 打印机存在一个严重的硬件Bug：连续双面打印时必然在第二页卡纸！**

这是一个困扰用户多年的设备缺陷，具体表现为：
- ✗ 打印机的自动双面功能只能一页一页地打印双面
- ✗ 只要连续发送两个或更多双面打印任务，第二页必定卡纸
- ✗ 无法通过驱动设置或固件更新解决
- ✗ 打印多页文档时需要人工干预，每打印一页就要等待再继续

### 其他类似问题
许多老旧型号的打印机也存在类似问题：
- 打印机处理速度慢，连续发送任务时页面读取失败
- 机械进纸机构老化，需要间隔时间避免卡纸
- 需要手动分批打印，操作繁琐且低效

### 本脚本的解决方案
**通过软件层面的时间控制，绕过硬件缺陷，实现稳定的连续双面打印**：

1. 将PDF按双面拆分成多个小文件（每个2页）
2. 逐个发送打印任务
3. 每次打印后强制等待15秒（或可配置时间）
4. 让打印机完全完成当前双面任务后再开始下一个
5. **从软件层面彻底避免连续双面打印导致的卡纸问题**

## 💡 设计原理

### Brother MFC-7480D 卡纸问题分析

**问题根源**：
- 打印机的双面打印机械结构存在设计缺陷
- 当第一页双面打印完成后，如果立即接收第二个双面任务，进纸机构还没有完全复位
- 导致第二页进纸时卡在半路，触发卡纸错误
- **这是硬件层面的时序问题，无法通过驱动或设置修复**

**解决思路**：
既然硬件无法修复，就从软件层面控制打印时序：
1. **强制延迟**：在两个双面打印任务之间插入足够的等待时间
2. **分批发送**：将大文档拆分成多个小任务，避免打印机自动连续处理
3. **状态同步**：确保打印机完全完成当前任务后再发送下一个

### 工作流程
```
原始PDF → 按双面分割 → 逐批打印 → 延迟等待 → 下一批 → 清理临时文件
  (N页)    (N/2个文件)   (2页/次)   (15秒)     (循环)     (自动)
                                      ↑
                              关键：给打印机足够
                              的机械复位时间
```

### 技术要点
1. **智能分割**: 使用 `pypdf` 库按页拆分PDF，每2页一组（对应一张纸的正反面）
2. **批量打印**: 调用 Adobe Acrobat Reader 的命令行接口进行打印
3. **延迟控制**: 每批打印后强制等待15秒（针对MFC-7480D优化），确保打印机机械结构完全复位
4. **双面配置**: 自动配置打印机为双面打印模式（长边/短边可选）
5. **自动清理**: 临时文件存放在系统临时目录，打印完成后自动删除

### 为什么15秒？
经过实际测试：
- **10秒**：仍有小概率卡纸
- **15秒**：已测试连续打印81张双面（161页PDF）无卡纸
- **20秒**：更保险，适合打印机状态不佳时使用

## 🚀 使用方法

### CLI 命令行模式

#### 基本用法
```bash
# 使用默认打印机，15秒延迟
python cli_app.py document.pdf

# 指定延迟时间
python cli_app.py document.pdf -d 20

# 指定打印机（支持模糊匹配）
python cli_app.py document.pdf -p Brother

# 按编号选择打印机
python cli_app.py document.pdf -p 1

# 短边翻转双面打印
python cli_app.py document.pdf --duplex short
```

#### 辅助命令
```bash
# 列出所有可用打印机（带编号）
python cli_app.py -l

# 查看指定打印机当前配置
python cli_app.py -i -p Brother

# 查看默认打印机配置
python cli_app.py -i

# 查看完整帮助
python cli_app.py --help
```

#### 完整参数说明
```
位置参数:
  pdf_file              要打印的PDF文件路径

可选参数:
  -d, --delay N         每批打印后的延迟秒数（默认: 15）
  -p, --printer NAME    打印机名称（支持模糊匹配如 'Brother'；或编号如 '1'）
  -l, --list            列出所有可用打印机
  -i, --info            显示打印机当前配置信息
  --duplex {long,short,none}
                        双面打印模式: long=长边翻转(默认), short=短边翻转, none=单面
  --keep-temp           保留临时文件（用于调试）
  --no-config           跳过自动双面打印配置
```

#### 打印机选择方式
脚本支持三种打印机指定方式，解决不同系统下打印机名称差异问题：

1. **模糊匹配**：输入打印机名称的一部分，自动匹配
   ```bash
   python cli_app.py doc.pdf -p Brother     # 匹配所有含"Brother"的打印机
   python cli_app.py doc.pdf -p MFC         # 匹配所有含"MFC"的打印机
   ```
   如果匹配到多个，会进入交互式选择。

2. **数字编号**：使用 `-l` 查看编号后直接指定
   ```bash
   python cli_app.py -l                     # 查看编号
   python cli_app.py doc.pdf -p 1           # 选择第1号打印机
   ```

3. **交互式选择**：不指定 `-p` 参数时，会提示选择
   ```
   当前默认打印机: Brother MFC-7480D Printer
   使用默认打印机? (回车=是, n=选择其他):
   ```

## ⚙️ 配置建议

### 延迟时间设置
不同打印机建议的延迟时间：

| 打印机型号 | 建议延迟 | 说明 |
|-----------|---------|------|
| **Brother MFC-7480D** | **15-20秒** | **⚠️ 专门针对连续双面卡纸Bug优化<br>15秒已测试81张双面无卡纸<br>20秒更保险** |
| 激光打印机（高速） | 10-15秒 | 处理速度较快 |
| 喷墨打印机（低速） | 20-30秒 | 处理速度较慢 |
| 网络打印机 | 20-25秒 | 考虑网络传输延迟 |
| 其他老旧打印机 | 15-25秒 | 根据实际情况调整 |

### Brother MFC-7480D 特别说明
如果你的打印机有连续双面卡纸问题：
1. **首次使用**：建议从20秒开始测试
2. **稳定后**：可以逐步降低到15秒
3. **打印机老化**：如15秒仍偶尔卡纸，增加到20-25秒
4. **重要文档**：宁可慢一点，使用20秒确保万无一失

**实测数据**：
- ✅ 15秒间隔：成功打印161页PDF（81张双面），0次卡纸
- ✅ 20秒间隔：成功打印161页PDF（81张双面），0次卡纸
- ✗ 10秒间隔：偶尔会在第10-20张时卡纸

### 双面打印模式
| 模式 | 参数 | 说明 |
|------|------|------|
| **长边翻转** | `--duplex long`（默认） | 适合竖向文档，翻页像书本 |
| **短边翻转** | `--duplex short` | 适合横向文档，翻页像记事本 |
| **单面** | `--duplex none` | 仅单面打印 |

## 🔧 故障排除

### 问题1: Brother MFC-7480D 仍然卡纸
**可能原因**：延迟时间不够、打印机机械部件老化、纸张质量问题

**解决方案**：
1. **增加延迟时间**：将延迟从15秒增加到20秒或25秒
2. **检查纸张**：使用优质打印纸，避免受潮或卷曲的纸张
3. **清洁打印机**：清理进纸辊和双面单元
4. **小批量测试**：先打印10页测试，确认稳定后再打印大文档

### 问题2: 找不到打印机
**解决方案**：
```bash
# 列出所有可用打印机及编号
python cli_app.py -l
```
使用编号或更精确的名称片段指定打印机。注意不同系统下打印机名称可能不同。

### 问题3: 打印失败或跳页
**解决方案**：增加延迟时间（`-d` 参数），给打印机更多处理时间。

### 问题4: Adobe Reader未找到
**解决方案**：
- 安装 Adobe Acrobat Reader DC
- 脚本会自动从注册表和常见路径查找 Acrobat
- 找不到时会自动降级使用系统默认打印方法

### 问题5: 临时文件
**说明**：临时文件存放在系统临时目录（`%TEMP%\pdf_duplex_*`），打印完成后自动删除。如需调试，使用 `--keep-temp` 参数（CLI）或勾选「保留临时文件」（GUI）保留临时文件。

### 问题6: GUI 无法启动
**解决方案**：
- 确保系统安装了 tkinter（大多数 Python 安装已内置）
- Windows 系统通常无需额外安装

## 📌 注意事项

1. **Brother MFC-7480D 用户必读**：
   - ⚠️ 这个脚本是专门为解决连续双面卡纸问题设计的
   - 15秒延迟是经过实际测试的最佳值（已验证161页无卡纸）
   - 如果你的打印机老化严重，建议使用20秒
   - 不要试图缩短延迟时间，否则会重新出现卡纸问题

2. **Adobe Reader推荐**: 虽然脚本可以使用系统默认打印，但Adobe Reader能提供更好的页面缩放和双面打印支持

3. **延迟时间**: 首次使用建议从较大的延迟开始（如20秒），观察效果后再调整

4. **打印队列**: 确保打印机队列中没有积压的任务

5. **纸张充足**: 确保纸盒中有足够的纸张（至少计算页数÷2的张数）

6. **中断恢复**: 如果中途中断（Ctrl+C 或 GUI 取消），已发送的打印任务仍会继续执行

7. **时间成本**: 大文档打印需要较长时间，161页的文档约需20-25分钟，请做好心理准备

8. **成功率**: 严格按照推荐配置使用，成功率可达100%
